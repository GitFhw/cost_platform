# 项目说明

## 项目定位

`cost_platform` 是面向企业级核算场景的重构项目。

目标不是做单一场景工具，而是沉淀一套可支持多业务域、多合同、多费用项的核算平台底座。

## 当前技术底座

### 后端

- 基于若依 Spring Boot 3 体系
- JDK 17+
- 目录：`cost_platform_back`

### 前端

- 基于若依 Vue3 + Element Plus + Vite
- 目录：`cost_platform_front`

## 业务模型方向

后续按以下主线逐步收敛产品模型：

1. 场景
   - 可理解为合同、核算主题、业务方案
2. 费用
   - 某个场景下需要核算的具体费用项
3. 变量
   - 费用判断和计算依赖的影响因素
4. 规则
   - 某个费用在不同条件下的费率或公式
5. 阶梯
   - 某条规则下的阶梯费率明细
6. 发布
   - 将草稿配置沉淀为可运行、可追溯的版本快照

## 重构原则

- 以业务人员易懂、易维护为第一优先级
- 先把概念和维护路径理顺，再继续叠加复杂能力
- 基础字典使用若依现有字典体系承接
- 核算主数据与规则使用独立业务表承接

## 第一阶段范围

### 1. 系统字典

- 独立梳理核算平台所需基础字典
- 业务域改造成字典下拉

### 2. 场景中心

- 明确“场景”的业务含义
- 强化业务域筛选和分类
- 先接通真实的 `cost_scene` 表与标准 CRUD 链路

### 3. 费用中心

- 对齐老项目“基础费目维护”的定位
- 先建立“场景 -> 费用”的主数据关系
- 后续规则与阶梯都围绕费用继续展开

## 当前已落地内容

### 1. 业务域字典

- 已规划 `cost_business_domain` 作为核算平台统一业务域字典
