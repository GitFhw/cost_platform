# 工作上下文改造清单

## 背景

当前核算平台在场景、账期、版本三个关键维度上，已经有部分页面跳转透传能力，但整体仍然偏分散：

- 不同页面各自维护 `billMonth` 默认值
- 有的页面走路由参数，有的页面回落当前月
- 场景切换后，版本和账期的默认行为不一致
- 关键页仍然允许自由手填账期，容易出现格式错误和上下文断裂

这会导致业务用户在“数据接入 -> 试算 -> 正式核算 -> 结果台账 -> 账期治理”之间切换时，经常需要重复确认自己当前到底在操作哪个场景、哪个账期、哪个版本。

## 改造目标

本轮改造先收口“工作上下文”，统一三个核心状态：

- `sceneId`
- `billMonth`
- `versionId`

原则如下：

1. 页面首次进入时，优先继承当前工作上下文。
2. 页面间跳转时，继续尊重路由参数透传。
3. 用户在核心业务页修改场景、账期、版本后，自动同步回工作上下文。
4. 账期尽量改成月份选择器，减少手填格式错误。
5. 不做“所有页面强绑定同一个账期”的硬同步，保留每页临时覆盖能力。

## 本轮落地范围

### 已完成

- 新增统一工作上下文工具：
  - `cost_platform_front/src/utils/costWorkContext.js`
- 兼容旧场景上下文存储：
  - `cost_platform_front/src/utils/costSceneContext.js`
- 接入核心页面默认带入与同步：
  - 数据接入
  - 试算中心
  - 正式核算
  - 结果台账
  - 告警中心
  - 账期管理
- 关键账期输入改为月份选择器，减少手工输入 `yyyy-MM`

### 本轮暂不处理

- 按场景联动筛选“可用账期列表”
- 全局顶部工作上下文条
- 跨页显式提示“当前正在操作的场景/账期/版本”
- 基于场景自动推荐生效版本

## 下一轮建议

1. 在顶部增加“当前工作上下文”展示条。
2. 正式核算、结果台账、账期管理三页增加“同步为当前工作上下文”显式提示。
3. 将账期从自由输入进一步升级为“按场景可选账期”。
4. 在页面切换时增加轻量提示，避免用户误把历史查询上下文带入正式提交。
