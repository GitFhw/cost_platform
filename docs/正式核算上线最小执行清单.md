# 正式核算上线最小执行清单

更新时间：2026-04-02

## 使用说明
- 这份清单只保留当前真正阻塞上线的最小集合。
- 执行结果仍以 [`go_live_regression_execution_template.xlsx`](/D:/Desktop/cost_platform/docs/go_live_regression_execution_template.xlsx) 为正式台账。
- 建议执行顺序：
  1. 先做发布准备
  2. 再做正式核算主链
  3. 再做分片恢复和结果追溯
  4. 最后做权限与环境验证

## 阻塞上线最小集合

### 一、发布准备
- `GLR-001` 选择目标工作场景，确认场景状态、账期和默认版本正确
- `GLR-002` 完成一次正式发布，确认版本号、发布时间、发布人和快照摘要可见
- `GLR-003` 校验影响因素、规则、价格方案、变量配置均进入发布快照，且详情抽屉可查看

### 二、正式核算主链
- `GLR-004` 使用 `JSON 直传` 提交单笔正式核算任务，并生成结果
- `GLR-005` 使用 `JSON 直传` 提交批量输入，确认任务明细、结果台账和追溯台账正常生成
- `GLR-006` 创建导入批次，确认批次台账可查看批次摘要、样例明细和状态
- `GLR-007` 从导入批次台账回填批次继续提交正式核算任务，确认任务正确引用 `sourceBatchNo`

### 三、分片恢复
- `GLR-008` 提交一笔足够触发多分片的正式核算任务，确认分片列表、范围、耗时和成功失败数正确
- `GLR-009` 制造至少一条失败明细，确认失败聚合摘要和高频错误原因展示正确
- `GLR-010` 对失败分片执行一次重试，确认分片状态、任务统计和任务明细同步刷新

### 四、结果追溯
- `GLR-015` 按场景、账期、任务过滤结果，确认金额、对象标识和执行版本正确
- `GLR-016` 打开结果详情和追溯详情，确认规则命中、变量快照和解释链路可查看
- `GLR-017` 校验导入批次任务与 JSON 任务在结果台账中的查询口径一致

### 五、权限与环境验证
- `GLR-018` 管理员可在左侧菜单看到导入批次、正式核算、结果台账、告警中心
- `GLR-019` 普通角色在未授权 `cost:task:list`、`cost:alarm:list` 等权限时，不应看到对应菜单与操作按钮
- `GLR-021` 目标环境执行 Flyway 后，确认 `V20260402_012 / 013 / 014` 迁移按顺序成功

- `接近上线，可进入上线前验收阶段`

切换为：
