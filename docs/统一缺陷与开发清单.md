# 统一缺陷与开发清单

## 1. 使用说明

从本文件起，项目内分散的缺陷、遗留项、风险、临时建议和阶段审计结论统一收口到这一份文档。

后续判断：

- 什么已完成
- 什么仍阻塞上线
- 什么还可以继续开发
- 下一步优先做什么

都以本文件为唯一维护入口。

状态口径：

- `已完成`：当前阶段代码、页面、接口和自动化证据已经闭环
- `待人工回归`：功能已具备，但还缺页面联调或人工回归留痕
- `待环境验证`：本地与测试已通过，但还缺目标环境验证
- `未完成`：当前仍未落地
- `后续增强`：不阻塞当前阶段，但仍是明确待办

## 3. 当前严格结论

### 3.0 2026-04-29 评审补充结论

本次将 `cost/dev` 快进合并到 `main` 后，对代码、建设文档、需求矩阵、上线回归模板和历史问题清单做了一轮复核。当前判断如下：

- 当前版本的主要矛盾已经不是“核心功能未开发”，而是“上线验收证据、目标环境压测证据和真实业务人工回归证据尚未闭环”。
- 正式上线包会移除 Flyway 相关代码与运行期迁移链路，因此 Flyway 不再作为生产上线阻塞项；目标环境放行改为校验上线初始化脚本执行后的库表、菜单、字典、权限与基础数据结果。
- 已具备上线验收基础的主链包括：场景、费用、变量、规则、公式、发布、试算、正式核算、导入批次、分片调度、结果台账、告警中心、第三方接入与开放接口。
- 下一阶段若继续做可编码开发，优先级应低于上线证据闭环；除非发现 P0 回归失败，否则不建议继续插入大功能。
