# 统一缺陷与开发清单

## 1. 使用说明

从本文件起，项目内分散的风险清单、遗留项跟踪、缺口核查、临时建议清单和单次审计结论统一收口到这一份文档。  
后续判断“是否完成、是否阻塞上线、下一步做什么”，以本文件为唯一维护入口。

状态口径：

- `已完成`：当前阶段代码、页面和自动化证据已闭环
- `待人工回归`：功能已具备，但仍需页面/联调实际操作验证
- `待环境验证`：功能已具备，但仍需目标环境验证
- `未完成`：当前仍未落地
- `后续增强`：不阻塞当前阶段，但仍是明确待办

## 6. 当前开发顺序

严格按当前结论，后续顺序如下：

1. 所有缺陷、建议和页面差异只维护这一份清单。
2. 若继续推进 `P0`，重点不是补新功能，而是完成目标环境验证和人工回归留痕。
3. 若继续做可编码开发，优先顺序为：
   1. `DIFF-001` 版本差异默认选中逻辑调整
   2. `PUBLISH-006` 发布检查结果沉淀与回看收口
   3. `PAGE-002` 独立发布审计页
   4. `SCENE-006` 场景中心发布治理摘要
   5. `PAGE-001` 场景适用组织树下拉
   6. `PAGE-003` / `PAGE-004` 结果与审计导出
