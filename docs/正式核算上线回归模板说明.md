# 正式核算上线回归模板说明

## 目的
这份模板用于把当前阻塞上线的“人工回归执行结果”收敛到统一口径，避免回归过程里出现：

- 只记录通过/失败，不留证据
- 发现问题后没有缺陷单号或责任动作
- 同一个检查项在多人执行时字段口径不一致

## 模板文件
- `docs/go_live_regression_execution_template.xlsx`

## 生成脚本
- `scripts/generate_go_live_regression_template.py`

## 推荐使用方式
1. 先在目标环境执行上线初始化 SQL，并完成基础菜单、字典、权限刷新。
2. 再按模板中的 `ExecutionLedger` 顺序逐条执行。
3. 每条至少填写：
   - `执行状态`
   - `执行人`
   - `执行日期`
   - `证据链接/截图说明`
   - `结果说明`
4. 若发现问题，补充：
   - `缺陷单号`
   - `下一步动作`
5. 所有 `P0 + 阻塞上线=是` 的行都为“已通过”后，才可以进入最终上线确认。

## 证据清单生成

统一上线回归脚本会在输出目录同步生成 `manual-evidence-checklist.md` 和 `manual-evidence-checklist.json`，用于把 `GLR-*` 最小放行项、执行人、环境、版本和附件目录索引收敛到同一个证据入口。

示例：

```powershell
.\cost_platform_back\bin\run-go-live-regression.ps1 `
  -MysqlHost 127.0.0.1 `
  -MysqlPort 3306 `
  -MysqlUser root `
  -MysqlPassword "<password>" `
  -Database cost_platform `
  -Executor "张三" `
  -EnvironmentName "UAT" `
  -BuildVersion "v0.3.0" `
  -AttachmentDir "D:\release-evidence\screenshots"
```

附件文件名中包含 `GLR-001`、`GLR-002` 等用例编号时，脚本会自动把文件索引到对应检查项，方便再回填到正式 Excel 台账。

## 当前模板覆盖范围
- 发布快照准备
- JSON / 导入批次双模式正式核算
- 分片监控、分片重试、失败恢复
- 告警中心与缓存治理联动
- 结果台账与追溯解释
- 菜单权限与普通角色权限回归
- 目标环境上线初始化验证
- 前后端构建校验

## 后续建议
- 每次补新的阻塞上线项时，优先同步更新模板和生成脚本。
- 如果后面要形成正式验收包，可以在模板基础上再补一个“验收结论”页签。
