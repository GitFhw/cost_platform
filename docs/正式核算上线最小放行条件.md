# 正式核算上线最小放行条件

更新时间：2026-04-02

## 目标
这份文档只回答一个问题：当前版本在什么条件下，才可以从“接近上线”切换到“可上线运行”。

## 放行前必须全部满足

1. 目标环境 Flyway 已执行成功
   - 重点确认 `V20260402_012__thread5_task_partition.sql`
   - 重点确认 `V20260402_013__thread5_input_batch.sql`
   - 重点确认 `V20260402_014__thread5_task_batch_menu.sql`
   - 校验方式：
     - 审计页“上线校验总览”对应项为绿色
     - 或数据库 `flyway_schema_history` 中存在成功记录

2. 关键库表已经在目标库存在
   - `cost_calc_task_partition`
   - `cost_calc_input_batch`
   - `cost_calc_input_batch_item`

3. 关键菜单已经在目标环境可见
   - `正式核算`
   - `导入批次`
   - `结果台账`
   - `告警中心`

4. 结构化回归台账中所有 `P0 + 阻塞上线=是` 的用例都为“已通过”
   - 台账文件：[`go_live_regression_execution_template.xlsx`](/D:/Desktop/cost_platform/docs/go_live_regression_execution_template.xlsx)

5. 至少完成一轮真实联调回归
   - 发布快照准备
   - `JSON 直传` 正式核算
   - `导入批次` 正式核算
   - 分片监控 / 分片重试 / 失败恢复
   - 结果台账 / 追溯解释
   - 告警中心 / 任务回跳
   - 管理员 / 普通角色权限验证

6. 构建校验保持通过
   - 后端：`mvn -q -DskipTests package`
   - 前端：`npm run build:prod`

## 不阻塞本次上线但必须挂牌跟踪

- 告警通知中心接入
- 自动化恢复策略
- 分布式调度与分布式锁
- 百万级压测报告
- 第三方变量真实执行链
- 公式实验室版本管理与模板库

## 当前结论口径

满足以上所有条件前：
- 当前状态：`接近上线，可进入上线前验收阶段`

全部满足后：
- 当前状态：`可上线运行`
