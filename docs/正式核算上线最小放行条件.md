# 正式核算上线最小放行条件

更新时间：2026-04-29

## 目标
这份文档只回答一个问题：当前版本在什么条件下，才可以从“接近上线”切换到“可上线运行”。

## 放行前必须全部满足

1. 目标环境上线初始化已完成
   - 正式上线包会移除 Flyway 相关代码与运行期迁移链路。
   - 上线验收不再以 `flyway_schema_history` 作为放行依据。
   - 放行依据改为目标库已按上线初始化脚本完成建表、菜单、字典、权限与必要基础数据初始化。
   - Flyway migration 目录仅作为开发、演示、历史追溯和初始化脚本生成参考，不作为生产启动依赖。

2. 关键库表已经在目标库存在
   - `cost_calc_task_partition`
   - `cost_calc_input_batch`
   - `cost_calc_input_batch_item`

3. 关键菜单已经在目标环境可见
   - `正式核算`
   - `导入批次`
   - `结果台账`
   - `告警中心`

4. 结构化回归台账中所有 `P0 + 阻塞上线=是` 的用例都为“已通过”
   - 台账文件：[`go_live_regression_execution_template.xlsx`](/D:/Desktop/cost_platform/docs/go_live_regression_execution_template.xlsx)

5. 至少完成一轮真实联调回归
   - 发布快照准备
   - `JSON 直传` 正式核算
   - `导入批次` 正式核算
   - 分片监控 / 分片重试 / 失败恢复
   - 结果台账 / 追溯解释
   - 告警中心 / 任务回跳
   - 管理员 / 普通角色权限验证

6. 构建校验保持通过
   - 后端：`mvn -q -DskipTests package`
   - 前端：`npm run build:prod`

## 不阻塞本次上线但必须挂牌跟踪

- 目标环境百万级压测报告与长稳压测证据
- 真实业务人工回归证据包沉淀
- 大 Service 与超大前端页面拆分治理
- 前端生产包体积与路由级拆包治理
- 变量缓存刷新真实链路深化
- 公式实验室复杂公式集压测与模板库增强

## 当前结论口径

满足以上所有条件前：
- 当前状态：`接近上线，可进入上线前验收阶段`

全部满足后：
- 当前状态：`可上线运行`
