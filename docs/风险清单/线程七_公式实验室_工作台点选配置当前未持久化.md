# 线程七：公式实验室工作台点选配置当前未持久化

## 所属线程
线程七：公式实验室与表达式资产化

## 当前现状
- 公式实验室前端已经支持业务人员点选变量、条件、模板和函数，自动生成中文公式与标准表达式。
- 公式保存时当前仍以 `businessFormula` 和 `formulaExpr` 为主，工作台内部的点选向导结构没有单独持久化到后端。

## 风险描述
- 业务人员以“点选向导”方式创建公式后，二次编辑时可以继续编辑公式资产，但当前会优先回到专家模式，不能完整还原之前的点选结构。
- 如果后续要支持更复杂的公式模板、拖拽编排和长期维护，缺少工作台配置持久化会影响编辑体验和治理一致性。

## 建议处理
- 在线程七后续整改中补齐 `builder_config_json` 或等价正式字段。
- 让公式工作台在保存、发布快照、追溯解释中同时保留：
  - 中文业务公式
  - 标准表达式
  - 点选构建配置

## 验收口径
- 已保存的公式再次打开时，工作台能够完整恢复：
  - 点选模式
  - 模板类型
  - 条件列表
  - 区间列表
  - 结果配置
