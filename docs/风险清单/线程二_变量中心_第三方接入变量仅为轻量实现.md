# 风险记录

- 线程：线程二
- 功能：变量中心 / 第三方系统接入变量
- 风险等级：高

## 描述

当前线程二虽然已经提供“测试接口、预览数据、刷新缓存”的轻量入口，但和详细设计、页面原型、高级设计文档要求的生产级接入能力仍有明显差距。

已识别差距：

- 未形成正式的来源系统标识、认证方式、密钥托管模型。
- 未形成字段映射与值转换规则的持久化配置。
- `previewRemoteData` 仍返回演示样例行，并非真实第三方数据装配链路。
- “刷新缓存”仅返回轻量提示，未接入真实缓存刷新、同步状态管理、失败重试和告警。
- 尚未支持实时、准实时、定时同步等正式拉取模式。

## 影响

- 当前能力更适合演示和页面联调，不适合直接作为真实生产接入方案。
- 如果后续运行链直接依赖当前实现，容易造成变量来源口径与正式计费链路不一致。

## 建议

- 后续整改时严格以详细设计和高级核算引擎设计为准，补齐来源系统、鉴权、字段映射、同步模式、缓存和失败治理模型。
- 在线程五/六接入运行链时，禁止继续把演示预览逻辑当成正式数据接入逻辑。
