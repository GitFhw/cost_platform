# 线程七：公式实验室变量规则运行链尚未全链强制按公式编码执行

## 风险描述
- 当前线程七已把变量中心、规则中心和运行链接上 `formulaCode` / `amountFormulaCode`。
- 但为了兼容已存在数据和前序线程实现，变量公式与金额公式仍保留“直接表达式兜底”能力。
- 这意味着当前平台还处在“公式编码优先、原始表达式兼容”的过渡阶段，而不是完全强制按公式资产执行。

## 当前影响
- 线程七可以先平滑落地，不会阻断已有数据使用。
- 但如果长期保留双轨，后续治理、审计、快照和追溯会继续存在口径分叉风险。

## 后续计划
- 建议在线程八统一收口变量、规则、阶梯、试算和正式核算的公式引用模式。
- 后续新增配置优先只允许公式编码，原始表达式仅作为历史兼容迁移能力保留。
