# 线程七：全局场景上下文当前仅覆盖公式实验室与运行链关键页

## 所属线程
线程七：公式实验室与表达式资产化

## 当前现状
- 公式实验室、试算中心、正式核算已经支持记住最近一次所选场景。
- 用户在这些页面切换场景后，查询区和工作区会同步刷新对应变量、版本和输入模板。

## 风险描述
- 费用中心、变量中心、规则中心、发布中心、结果台账等页面尚未统一接入同一套全局场景上下文。
- 用户跨页面操作时仍可能重复选择场景，离“全页面场景控制”还差一步。
- 如果后续每个页面继续各自维护场景状态，容易出现体验不一致和筛选口径漂移。

## 建议处理
- 在线程八继续推广统一的场景上下文工具，覆盖费用、变量、规则、发布和结果页面。
- 保持“可选全局场景”的策略：
  - 用户显式选择全局场景时，相关页面默认按该场景过滤。
  - 用户未选择时，仍保留各页面独立查询能力。

## 验收口径
- 用户在任一中心页面选择场景后，切换到其他关键中心页面时默认继承该场景。
- 页面仍保留手动切换和清空场景能力，不影响独立查询。
- 变量、规则、发布、试算、正式核算和结果台账的场景口径保持一致。
