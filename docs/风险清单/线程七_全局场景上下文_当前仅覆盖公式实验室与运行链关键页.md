# 线程七：全局场景上下文当前仅覆盖公式实验室与运行链关键页

## 所属线程
线程七：公式实验室与表达式资产化

## 当前现状
- 公式实验室、试算中心、正式核算、费用中心、变量中心、规则中心、发布中心、结果台账已经支持记住最近一次所选场景。
- 场景中心已经支持设置和清除“当前工作场景”，用户可以在场景台账中直接指定全局默认场景。
- 用户在这些页面切换场景后，查询区和工作区会同步刷新对应变量、费用、版本和输入模板。

## 风险描述
- 当前主链页面已经完成第一轮全局场景上下文接入，原风险已基本关闭。
- 后续如果再新增中心页面或新工作台，仍需继续复用同一套场景上下文工具，避免新增页面重新走回各自维护状态的老路。

## 建议处理
- 保持当前实现作为统一基线，新页面继续复用同一套场景上下文工具。
- 保持“可选全局场景”的策略：
  - 用户显式选择全局场景时，相关页面默认按该场景过滤。
  - 用户未选择时，仍保留各页面独立查询能力。

## 验收口径
- 用户在任一主链中心页面选择场景后，切换到其他关键中心页面时默认继承该场景。
- 场景中心支持设置和清除当前工作场景，页面仍保留手动切换和清空场景能力，不影响独立查询。
- 费用、变量、规则、发布、试算、正式核算和结果台账的场景口径保持一致。
