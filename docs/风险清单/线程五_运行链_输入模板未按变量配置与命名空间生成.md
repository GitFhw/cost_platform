# 风险记录

- 线程：线程五
- 功能：试算 / 正式核算输入模板
- 风险等级：高
- 当前状态：已完成本轮整改，保留为关闭记录

## 描述

线程五运行页原先的输入方式，和详细设计、高级设计文档以及 `charge` 版本体现出的命名空间上下文思想存在偏差。

已识别问题：

- 输入示例曾采用硬编码字段表达，未严格按已发布变量配置动态生成。
- 页面没有显式体现命名空间上下文，例如 `zx.xxx`、`sg.xxx` 这类分域输入口径。
- 容易让人误解为“一个场景只对应一个简单 JSON 输入”，而不是“按场景装配统一上下文，再逐个费用计算”。

## 影响

- 容易造成业务对产品计费方式的理解偏差。
- 如果后续直接沿用当前输入表达，运行链将偏离变量中心和第三方映射的正式口径。

## 本轮整改结果

- 已改为按发布快照和变量元数据生成输入模板。
- 已根据变量 `dataPath` / `variableCode` 生成命名空间结构，支持 `zx.xxx`、`sg.xxx` 等路径。
- 已在试算中心、正式核算页面展示模板说明和字段清单，替代硬编码示例。

## 后续观察点

- 运行链输入模板这项高风险本轮已关闭，不再作为当前阻塞项。
- 如果后续需要扩展到更多页面的全局场景上下文协同，请参考线程七新增风险文件。

## 建议

- 保持“输入模板基于发布快照和变量元数据生成”的统一口径。
- 后续继续把全局场景上下文扩展到更多中心页面，减少跨页面重复切换场景。
