# 正式核算上线回归清单

## 1. 发布快照准备
- 选择目标工作场景，确认场景状态、账期和默认版本正确。
- 在发布中心完成一次正式发布，确认版本号、发布时间、发布人和快照摘要可见。
- 校验影响因素、规则、价格方案、变量配置均已进入发布快照，且详情抽屉可查看。

## 2. 正式核算提交流程
- 使用 `JSON 直传` 提交一笔正式核算任务，确认任务可进入任务中心并生成结果。
- 使用 `JSON 直传` 提交一组批量输入，确认任务明细、结果台账和追溯台账正常生成。
- 使用 `导入批次` 创建批次，确认批次台账可查看批次摘要、样例明细和状态。
- 从导入批次台账回填批次继续提交正式核算任务，确认任务正确引用 `sourceBatchNo`。

## 3. 分片与失败恢复
- 提交一笔足够触发多分片的正式核算任务，确认任务详情展示分片列表、范围、耗时和成功失败数。
- 制造至少一条失败明细，确认任务详情展示失败聚合摘要和高频错误原因。
- 对失败分片执行一次 `重试分片`，确认分片状态、任务统计和任务明细同步刷新。
- 对失败明细执行一次 `重试`，确认兼容链路仍可用，且不会覆盖其他分片结果。

## 4. 告警与治理
- 从任务中心进入告警中心，确认自动带入 `sceneId`、`billMonth`、`taskId` 和未关闭状态。
- 任务失败或重试超限时，确认告警台账能看到对应记录，并可执行确认、关闭。
- 校验运行快照缓存状态可查询，缓存刷新操作有权限控制且执行后仍以数据库为准。

## 5. 结果与追溯
- 在结果台账按场景、账期、任务过滤结果，确认金额、对象标识和执行版本正确。
- 打开结果详情和追溯详情，确认规则命中、变量快照和解释链路可查看。
- 校验导入批次任务与 JSON 任务在结果台账中的查询口径一致。

## 6. 权限与菜单
- 管理员可在左侧菜单看到 `导入批次`、`正式核算`、`结果台账`、`告警中心`。
- 普通角色在未授予 `cost:task:list`、`cost:alarm:list` 等权限时，不应看到对应菜单与操作按钮。
- 校验任务取消、分片重试、告警确认/关闭、缓存刷新均受按钮权限控制。

## 7. 上线前构建校验
- 后端执行 `mvn -q -DskipTests package` 通过。
- 前端执行 `npm run build:prod` 通过。
- 确认目标环境已按上线初始化 SQL 完成库表、菜单、字典、权限和必要基础数据初始化。

## 8. 已知上线前缺口
- 当前仍缺全局任务/分片趋势告警视图和自动通知中心。
- 导入批次目前已具备任务中心与独立台账入口，但尚未扩展到更完整的运维报表。
- 真正百万级正式核算仍建议继续补输入源分页装载、分片级恢复台账和更强的批量写入兜底。
